## 목표

`useMyApplications()` · `usePastMeetings()` 훅 내부를 `GET /api/v1/me/sessions` 계열 API 호출로 교체하고,
모임 평가 제출을 `POST /api/v1/sessions/{slotId}/consumption-records`로 연결한다.
화면 컴포넌트 shape은 그대로 유지한다.

---

## API 대조 (Swagger ↔ api-schema.md ↔ 훅 ↔ UI)

| 엔드포인트                                           | 스키마 확인                                                             | 훅                                      | UI 소비               |
| ---------------------------------------------------- | ----------------------------------------------------------------------- | --------------------------------------- | --------------------- |
| `GET /api/v1/me/sessions`                            | `MySessionListResponse.sessions: MySessionItemResponse[]` ✅            | `useMyApplications` · `usePastMeetings` | `MeetingsScreen`      |
| `GET /api/v1/me/sessions/{slotId}`                   | `MySessionDetailResponse` ✅                                            | `usePastMeeting(id)`                    | `pastApplication.tsx` |
| `POST /api/v1/sessions/{slotId}/consumption-records` | `CreateConsumptionRecordRequest` / `CreateConsumptionRecordResponse` ✅ | `usePastEval.markEvaluated`             | `pastEval.tsx`        |

**SlotStatus 분기**:

- `OPEN` | `MENU_PROPOSED` → 신청 내역(useMyApplications)
- `COMPLETED` → 지난 모임(usePastMeetings)
