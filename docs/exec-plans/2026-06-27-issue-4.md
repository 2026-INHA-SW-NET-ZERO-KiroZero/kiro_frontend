## 목표

PRD §5의 더미데이터 배열을 타입과 함께 `src/data/`에 시드하고, UI가 직접 import하지 않도록
커스텀 훅으로 감싼다. 백엔드가 붙으면 훅 내부만 교체하면 되도록 UI ↔ 데이터 경계를 훅으로 분리.

## 데이터 출처 (1:1, 추측 금지)

- 값/카피/이모지: `design_handoff_kirozero/design/KiroZero.dc.html`의 `class Component` 생성자 블록
  (라인 1454~1554) — `me`, `allergyAll`, `rooms`(생성 로직), `aggIngredients`, `menuCandidates`,
  `voteMenus`, `decidedMenu`, `notifData`, `myApplications`, `settlementData`, `pastMeetings`,
  `partyPool`(라인 1699~1704).
- shape 정의: `FRONTEND_PRD.md §5`.

## 범위 조정 (이슈 task 목록 vs 실제 소스)

- **`baseIngredients`** — dc.html/PRD §5에 존재하지 않음 → 생성 안 함. (room의 `baseCount`로 대체됨)
- **`adminRoomsData`** — PRD §0·§8에서 **운영자(Admin) 기능 제거** → 생성 안 함(재도입 금지).
- 위 2개를 제외한 §5의 12개 구조를 모두 시드한다.

## 관련 파일 (신규)

- `src/types/` — 도메인별 타입 + barrel(`index.ts`) + `query.ts`(훅 반환 shape `DataResult<T>`).
- `src/data/` — 도메인별 시드 + barrel. dc.html 값 1:1.
- `src/hooks/` — `useMe`/`useAllergyOptions`/`useRooms`/`useRoomDetail`/`useAggIngredients`/
  `useMenuCandidates`/`useVoteMenus`/`useDecidedMenu`/`useNotifications`/`useMyApplications`/
  `useSettlement`/`usePastMeetings`/`usePastMeeting`/`usePartyPool` + `useSeed`(공용 헬퍼) + barrel.
- `src/theme/theme.ts` — 아바타 식별 색 팔레트(`avatarPalette`) + 누락 색 2종(`pink`,`leaf`) 추가.
  (참여자 아바타 색은 디자인 데이터지만 eslint가 theme 밖 hex를 막으므로 토큰으로 등록)
