## 4중 일치 분석 (Swagger ↔ api-schema ↔ 훅 타입 ↔ UI 소비)

### A. `useAllergyOptions` ← `GET /allergy-tags` — **부분 불일치**

- UI 소비(`my.tsx`·`editProfile.tsx`): `{a.emoji} {a.label}`.
- 백엔드: `tag`, `labelKo`, `description` — **`emoji` 없음**.
- `label ← labelKo`(정합). `emoji ← 백엔드 미제공`.
- 저장(PUT) 위해 `tag`도 필요 → `AllergyOption`에 `tag` 추가 권장.

### B. `useMe` ← `GET /auth/me` (`CurrentUserResponse`) — **필드 일부 불일치(개선됨)**

- `Me { name, dept, email, allergy, grade, gradeLv, cash, leaves }`
  vs `CurrentUserResponse { userId, email, nickname, cookingSkill, cash, allergyTags }`.
- 정합 매핑: `name ← nickname`, `email ← email`, `allergy ← allergyTags`(tag→labelKo, allergy-tags 조회 필요), **`cash ← cash`(🔄 신규 제공)**.
- **여전히 백엔드 미제공**: `dept`, `grade`, `gradeLv`, `leaves`.
- 현재 화면 실제 소비 필드: `my.tsx` → `me.name`, `me.leaves` / `useProfile` 초기화 → `me.allergy`.
  → `dept`·`grade`·`gradeLv`는 **현재 어떤 화면도 소비하지 않음**. `leaves`만 소비됨(백엔드 소스 없음).

### C. 프로필 저장 ← `PUT /me/profile` (`UpdateProfileRequest`)

- 요청 `nickname*` 필수인데 editProfile 화면은 닉네임을 편집하지 않음 → **현재 nickname을 그대로 전송**(useMe의 nickname).
- `skillLevel(상/중/하) → cookingSkill` = `SKILL_TO_SERVER`(존재).
- `allergies(한국어 라벨) → allergyTags(tag 문자열)` = allergy-tags의 `labelKo→tag` 역매핑 필요.
- 현재 "저장하기"는 `router.back()`만 함 → 비동기 PUT 호출 + 성공 시 useMe 캐시 갱신으로 변경.

### D. 누적 리포트 ← `GET /me/results/total` (`MyResultTotalResponse`) — **🔄 월별 연동 가능해짐**

- 제공(확장): 누적 `completedSessionCount`·`totalUsedGrams`·`totalEstimatedCarbonSavedKgco2e`·`totalRefundAmount`,
  참여/재료 stat `togetherPeopleCount`·`providedIngredientCount`·`usedIngredientCount`·`averageIngredientUseRate`,
  전월 대비 `currentMonth/previousMonth/monthOverMonthCarbonDeltaKgco2e`, `insightMessage`, **`monthlyResults[]`(월별 내림차순)**.
- `report.tsx`(useReport)가 그리는 값과의 매핑(이제 대부분 백엔드 제공):
  - 참여 모임 ← `completedSessionCount`, 함께한 사람 ← `togetherPeopleCount`,
    제공한 재료 ← `providedIngredientCount`, 사용한 재료 ← `usedIngredientCount`, 소진율(rate) ← `averageIngredientUseRate`.
  - eco 라인 카피 ← `insightMessage`(완전 일치), 탄소 환산값 ← `totalEstimatedCarbonSavedKgco2e`.
  - 히어로 "지난달보다 늘었어요" ← `monthOverMonthCarbonDeltaKgco2e` 부호.
  - 월별 바차트 ← `monthlyResults`의 `monthLabel` + 월별 `totalEstimatedCarbonSavedKgco2e`(또는 사용 그램).
- **여전히 미매핑**: 히어로 "음식물 절감 X kg"(`saved`) — 백엔드 `MonthlyResultSummaryResponse`엔 음식물 절감량(g) 필드가 없고 `totalUsedGrams`·탄소만 있음. "음식물 절감"의 정확한 소스 정의 필요.
- 단, **`useReport`는 이슈가 명시한 교체 대상(useMe·useAllergyOptions)에 포함되지 않음** → 리포트 연동은 별도 결정 필요(아래 D).

1. **알레르기 emoji 소스**: 백엔드 미제공. → `tag → emoji` 로컬 상수(프레젠테이션) 유지 후 `labelKo`와 결합 권장. (시드 파일은 삭제하되 emoji 매핑만 상수로 이전.) 동의?

3. **`me.leaves`(나뭇잎 보상)**: 백엔드에 **leaves 필드 없음**(확인됨). `cash`는 누적 환불 적립금으로 별개 개념 → 나뭇잎으로 임의 매핑 금지. 선택지: (a) `leaves`를 프론트 전용 placeholder로 유지, (b) my 화면 나뭇잎 슬롯을 `cash`(적립금) 표시로 바꿀지 — **제품 의도 확인 필요**.
4. **리포트(`useReport`) 연동 범위**: 백엔드가 월별 데이터를 제공하므로 연동 가능하나, 이슈가 명시한 교체 대상은 useMe·useAllergyOptions뿐. 선택지: (a) 이번 이슈 범위대로 useReport는 더미 유지하고 results/total은 `useResultTotal()` 신규 훅으로 노출만, (b) 범위 확장해 useReport까지 연동. "음식물 절감(saved) kg"의 백엔드 소스 미정의 문제도 함께 결정.
