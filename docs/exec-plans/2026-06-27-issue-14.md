# Issue #14 — 리포트 + MY + 프로필 수정 (§3.12–3.14)

## 범위 결정 (작업 중 확정)

- 이슈 #14는 **마일스톤 2(화면/디자인)**. 더미데이터 + 로컬 상태로 3화면을 픽셀 재현.
- 사용자 제안(`auth/me`로 프로필 조회)은 검토 결과 **별도 이슈로 분리**:
  - API 클라이언트 인프라(fetch 래퍼·base URL·인증 헤더)가 전혀 없음 → 신규 구축 필요.
  - 리포트는 `MyResultTotalResponse`가 누적 총합만 줘서(월별 추이·소진율·stat 없음) **백엔드로 재현 불가** → 더미 유지.
  - 프로필은 `GET /auth/me` + `PUT /me/profile` + `GET /allergy-tags`로 연동 가능(마일스톤 3).
- 공유 상태(요리 숙련도·알레르기)는 zustand 미설치 → 기존 `AuthContext` 패턴의 **Context**로 구현.
