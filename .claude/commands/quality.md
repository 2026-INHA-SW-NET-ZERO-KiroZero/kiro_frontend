# /quality — 품질 등급 재평가 워크플로우

현재 프로젝트 코드를 전체 분석해서 `docs/QUALITY_SCORE.md`를 업데이트한다.
각 영역별로 근거와 함께 등급(A~F)을 매기고, 이전 등급과 비교해 변화(↑↓→)를 표시한다.

## 평가 영역

- 컴포넌트 재사용성
- 네비게이션 (Expo Router 구조 일관성)
- API 연동 (현재는 더미데이터 → 훅 래핑 품질)
- 스타일링 일관성 (디자인 토큰 준수)
- 테스트 커버리지
- 접근성 (히트 타깃 ≥44px, 텍스트 대비)
- 타입 안전성 (`any` 비율, 타입 가드)
- 에러 핸들링 (에러/빈/로딩 UI)

분석 가능한 영역만 근거와 함께 등급을 매긴다. 초기에는 대부분 "미평가"일 수 있다.
