# /gc — 가비지 컬렉션 워크플로우

코드베이스의 드리프트를 전수 스캔하고 정리한다.
AI가 기존 코드를 보고 패턴을 따라하기 때문에 나쁜 패턴이 스노우볼된다. 이걸 정리한다.

## 1단계: 문서 ↔ 코드 동기화 점검

- `docs/FRONTEND.md` 패턴 vs 실제 코드
- `docs/DESIGN.md` 디자인 토큰 vs 실제 스타일
- `docs/generated/` (component-inventory, api-schema) vs 실제 파일
- 불일치 목록 + 자동 갱신 제안

## 2단계: 패턴 드리프트 스캔 (스노우볼 방지)

- 하드코딩 색상값(`#RRGGBB` 리터럴) 전수 검색 → 토큰 치환 제안
- `any` 타입, `console.log`, 인라인 스타일 전수 검색
- 공용 컴포넌트 미사용(중복 구현) 감지
- 더미데이터 직접 import(훅 미경유) 감지
- 위반 건수 + 자동 수정 가능 여부 표시

## 3단계: 데드 코드 정리

- 미사용 export, 미사용 파일, 미사용 의존성 감지
- 삭제 제안 (사용자 확인 후 실행)

## 4단계: 드리프트 방지 조치

- 반복 위반 → ESLint 규칙 승격 제안(harness-feedback 연계)
- 교정 지시 업데이트 필요 여부 확인

## 5단계: 보고서 + docs/ 갱신 + QUALITY_SCORE 업데이트

## 범위 밖

- 백엔드 코드, `node_modules`, 빌드 폴더(`.expo`, `dist`)
